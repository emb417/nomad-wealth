    üöÄ Usage Guide

    This guide explains how to run Nomad Wealth simulations, adjust settings, and understand the results.

    ‚ñ∂Ô∏è Running the Simulation

    1. Navigate to the project root Make sure you‚Äôre in the directory containing src/app.py.

    2. Run the entry point

      bash python src/app.py

    This command: - Loads your configuration files (buckets, balances, policies, taxes). - Prepares historical and future timelines. - Runs Monte Carlo trials in parallel to capture uncertainty. - Aggregates results into clear tables. - Generates interactive charts and CSV/HTML exports.

    1. Check the export/ directory

    All outputs (charts and CSVs) are saved here with timestamped filenames for easy tracking.

    ‚öôÔ∏è Controlling Behavior

    You can control how the simulation runs by adjusting flags in app.py:

    • Simulation Size ‚Üí SIM_SIZE sets the number of Monte Carlo trials.
    • Example Trials ‚Üí SIM_EXAMPLE_SIZE sets how many random trials are shown in detail (expenses, transactions, taxes, forecasts).
    • Chart Display ‚Üí SHOW_* flags decide which charts open interactively (e.g., net worth, examples, historical).
    • Chart Export ‚Üí SAVE_* flags decide which charts are saved to HTML/CSV.
    • Detailed Mode ‚Üí DETAILED_MODE forces all charts and exports to be generated for full transparency.

    üìÇ Inputs Required

    Nomad Wealth uses configuration files to define your plan:

    • profile.json ‚Üí birth month, retirement horizon and past MAGI.
    • buckets.json ‚Üí bucket definitions (must align with balance.csv).
    • balance.csv ‚Üí starting balances for each bucket.
    • policies.json ‚Üí rules for income, withdrawals, property, unemployment, Roth conversions.
    • tax_brackets.json ‚Üí IRS‚Äëaligned federal, state, payroll, capital gains, IRMAA, Medicare premiums.
    • inflation_rates.json ‚Üí baseline inflation + category profiles.
    • inflation_thresholds.json + gain_table.json ‚Üí asset class return regimes.
    • marketplace_premiums.json ‚Üí healthcare premiums.
    • fixed.csv ‚Üí one‚Äëtime events (e.g., tuition, travel).
    • recurring.csv ‚Üí ongoing monthly flows (e.g., insurance, food, utilities).

    All inputs must follow the schemas in Configuration Reference.

    üìà Outputs

    Nomad Wealth produces results you can explore in charts and exports:

    • Historical Charts ‚Üí bucket gains/losses and net worth trajectory.
    • Per‚ÄëTrial Example Charts ‚Üí detailed views of monthly expenses, transactions, taxes, and forecasts.
    • Aggregate Monte Carlo Charts ‚Üí distributions of returns, balances, taxes, withdrawals, and net worth.
    • CSV Exports ‚Üí tables of balances, tax breakdowns, monthly returns, and flow logs for reproducibility.

    üßæ Notes

    • Filenames include timestamps (YYYYMMDD_HHMMSS) for traceability.
    • Net worth = sum of all bucket balances each month.
    • SEPP rules enforce IRS 72(t) withdrawal compliance.
    • Roth conversions can occur before age 59.5 if configured.
    • Logging records export paths for transparency.
    • Detailed Mode ensures IRS‚Äëaligned reproducibility across all charts and exports.
